# Thompson Bill Gates Line

Thompson's 2018 essay drawing the canonical distinction between platforms and aggregators. Named after a Bill Gates quote (via Chamath Palihapitiya). Load-bearing for any strategic decision about whether to build a platform, build on a platform, build an aggregator, or build on an aggregator. The four answers have very different economic outcomes.

## The Defining Quote

> "A platform is when the economic value of everybody that uses it, exceeds the value of the company that creates it." (Bill Gates, via Chamath Palihapitiya)

The test: add up the value created by everyone building on top, plus the company itself. If third-party value exceeds the platform's value, it is a real platform. If not, it is just a product.

## Platform vs Aggregator

| Type | Mechanism | Value capture | Example |
|---|---|---|---|
| Platform | Facilitates third-party-to-user relationships | Minority of ecosystem value | Windows, AWS, Azure |
| Aggregator | Intermediates and controls user-supplier relationships | Majority of value | Google Search, Facebook |

Platforms distribute value across ecosystems. Aggregators concentrate it.

## Below the Line: App Stores

Apple's App Store is **below the Bill Gates Line**. Structurally a platform (enables third parties to build); economically an aggregator (Apple captures the majority of value via 30% rake plus distribution control).

The warning: enabling third parties to build does not make something a real platform. The economic test matters.

## Strategic Implications

Four choices for builders:

- **Build a real platform**: Strong network effects, but you capture less of the value pool. Need scale and longevity.
- **Build an aggregator**: You capture more value, but you must own the user relationship and never let third parties intermediate.
- **Build ON a platform** (AWS, Azure): Strong leverage. Captures value above the platform's slice. Recommended for most startups.
- **Build ON an aggregator** (Yelp on Google): Structurally disadvantaged. The aggregator can disintermediate you at any time.

The Yelp-on-Google example: even if Google featured Yelp prominently, users would eventually bypass Yelp. Building on an aggregator gives the aggregator the option to take your business.
